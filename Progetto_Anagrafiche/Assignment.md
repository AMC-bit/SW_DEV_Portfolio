1. Project Objective

Develop a system for managing records (Contacts and Clients) consisting of two client applications that communicate through REST APIs, sharing the same data on a SQL Server database.

There are no strict delivery deadlines: we evaluate quality, architectural correctness, and attention to detail rather than execution speed.

2. Technology Stack and General Rules
Framework: .NET 8 or .NET 9
Database: SQL Server
Data Access: Entity Framework Core in Database-First mode (using the Scaffold-DbContext command).
Important: Using Code-First with Migrations to create the database is not allowed.
External Libraries: Third-party NuGet packages may be used (e.g., CommunityToolkit.Mvvm for WPF, AutoMapper, etc.). If used, the reason for the choice must be documented in the README file.
Authentication: Not required. APIs and interfaces must be anonymously accessible.
3. Solution Architecture

The solution must contain (at least) the following projects:

Web App (Blazor Server)

Contains:

Web pages
REST Controllers exposed for the WPF client
Business logic
Database access through EF Core
Desktop App (WPF)

Desktop interface that must never access the database directly, but communicate exclusively through HTTP (JSON) requests to the Controllers in the Blazor project.

Data/Models Project (Class Library)

Contains EF Core models generated using Database-First.
This project must be referenced by both applications (Blazor and WPF) to share the same classes.

4. Data Model and SQL Script

The candidate must provide:

A SQL script (.sql) for database creation
A populated database backup (.bak)
Required Tables
Anagrafiche (Base Table)
Id (INT, PK, IDENTITY)
Nome, Cognome, Telefono, Email (NVARCHAR)
CodiceFiscale (NVARCHAR, UNIQUE)
PartitaIVA (NVARCHAR, UNIQUE)
IsPersonaFisica (BIT, NOT NULL, DEFAULT 0)
RagioneSociale (NVARCHAR, NULLABLE. Required if IsPersonaFisica = false)
AnagraficaContatto
AnagraficaId (INT, PK, FK to Anagrafiche.Id, ON DELETE CASCADE)
Note (NVARCHAR, NULLABLE)
Provenienza (NVARCHAR, NOT NULL. Allowed values: Website, LinkedIn, Advertising)
AnagraficaCliente
AnagraficaId (INT, PK, FK to Anagrafiche.Id, ON DELETE CASCADE)
CodiceCliente (INT, IDENTITY. Automatically generated by the DB)
DataRegistrazione (DATETIME, NOT NULL, DEFAULT '1900-01-01')
Note (NVARCHAR, NULLABLE)
Database Rules
BIT fields must have DEFAULT 0
DATETIME fields must have DEFAULT '1900-01-01'

Important: Correctly implement ON DELETE CASCADE on foreign keys of child tables.
Deleting an Anagrafica must physically remove related records as well (Hard Delete through referential integrity).

5. Business Logic and Features

The REST APIs in the Blazor controller must expose the following endpoints:

Contacts CRUD

Create, update, and delete an AnagraficaContatto
(which must also create the related record in the Anagrafiche table).

Clients CRUD

Create, update, and delete an AnagraficaCliente
(which must also create the related record in the Anagrafiche table).

Contact → Client Conversion

The user starts the conversion from the contact screen (after confirmation).

Logic:
The record in AnagraficaContatto is deleted
A new record is created in AnagraficaCliente
The new client record is linked to the same AnagraficaId
Base Anagrafica data remains unchanged
Unified Search
Input:
First Name
Last Name
Tax Code
VAT Number
Email
Output:

A unified list of Contacts and Clients.

Each row must clearly indicate whether the result is a Contact or a Client.

Each result must include an "Open Record" command to open the correct edit screen.

6. Data Validation (Technical Requirements)

Validation must occur on both:

Client side (WPF)
Server side (Blazor Controllers)
Validate() Method

The AnagraficaContatto and AnagraficaCliente classes
(or their extended/ViewModel classes) must contain a public Validate() method.

This method must return a list of strings containing validation errors, checking:

Required fields are not empty
Correct email format
Conditional rule: if IsPersonaFisica = false, RagioneSociale cannot be empty
Provenienza contains only allowed values for contacts
In Blazor Controllers (Server-Side)

Before every DB write operation (Create, Update, Delete), the controller must call Validate().

If validation errors exist:

The operation must be blocked
Errors must be returned to the WPF client with a clear message (HTTP 400 Bad Request)
In Blazor Controllers (Uniqueness)

The server must verify uniqueness of:

Tax Code (CodiceFiscale)
VAT Number (PartitaIVA)

before insert/update operations.

It must also handle EF Core exceptions (DbUpdateException) and return readable error messages.

In Blazor Controllers (Conversion)

The server must verify that a contact is not already a client before performing the conversion.
